# Дизайн-система проекта Quiz Platform

Этот документ описывает визуальный стиль, архитектуру интерфейса и правила UI/UX, которые лежат в основе проекта.

## 1. Концепция и Эстетика
Проект использует **Сложный Темный Глассморфизм (Sophisticated Dark Glassmorphism)**. 
Основные черты: полупрозрачные поверхности на фоне размытых цветовых пятен, строгая геометрия с плавными скруглениями, читабельный минимализм без лишних рамок.

## 2. Цветовая Палитра
Используется приглушенная палитра с элегантным контрастом:
*   `#22223b` — **Background** (Основной темный фон страницы)
*   `#4a4e69` — **Surface** (Плотные элементы, плашки, карточки)
*   `#9a8c98` — **Muted / Border** (Второстепенный текст, тонкие границы)
*   `#c9ada7` — **Accent** (Акцентный цвет: кнопки, выделение, подсветка)
*   `#f2e9e4` — **Text Primary** (Основной текст, яркий акцент)

**Дополнительные цвета для логики:**
*   `#6ba292` — **Success** (Правильный ответ)
*   `#d96c75` — **Danger** (Ошибочный ответ, ошибки)
*   `#d9a05b` — **Warning**

## 3. Типографика
- **Шрифт:** `Inter`, стандартные системные шрифты (Apple System, Roboto) как фолбэк.
- **Иерархия:** Четкое разделение заголовков (`h1` = 2rem+, `h2` = 1.5rem, `h3` = 1.25rem). Текст карточек и интерфейса (0.9rem - 1rem).
- **Свойства:** Плавный межстрочный интервал (`line-height: 1.6 - 1.8`), легкий трекинг для заголовков.

## 4. Архитектура Компонентов (UI Kit)
*   **Карточки (Cards):** Скругление `var(--radius) = 12px` или `20px` для крупных блоков. Легкая тень (`0 8px 24px rgba(0,0,0,0.25)`). При наведении — `transform: translateY(-2px)` и усиление тени.
*   **Стекло (Glass):** Полупрозрачный фон `rgba(34, 34, 59, 0.6)` с размытием заднего плана `backdrop-filter: blur(24px)`.
*   **Кнопки (Buttons):** 
    *   *Primary*: Цвет акцента, темный текст, при наведении — осветление.
    *   *Secondary*: Прозрачный фон с рамкой.
    *   *Ghost*: Без рамки и фона (например, для иконки шестеренки).
*   **Навигация (Dots):** Квадратные или слегка скругленные точки `34x34px`. Состояния: обычное, текущее (pulse-эффект), правильное (success), неправильное (danger).

## 5. Правила Верстки и Layout
*   **Сетки (CSS Grid):** Используется `grid-template-columns: repeat(auto-fill, minmax(280px, 1fr))` для адаптивного расположения карточек предметов. Зазоры (`gap`) от 20px до 24px.
*   **Мобильная адаптивность:**
    *   Обязательно строгое ограничение по ширине экрана: `max-width: 100vw`, `overflow-x: hidden`.
    *   Сжатие отступов (padding) с `24px` до `16px/20px` на мобильных экранах (`@media (max-width: 768px)`).
    *   Текст должен ломаться, если он длиннее экрана: `word-break: break-word`, `overflow-wrap: anywhere`.
*   **Медиа и Изображения:** 
    *   Картинки никогда не должны тянуться: `max-width: 100%`, `object-fit: contain`. 
    *   В вопросах и опциях максимальная высота ограничена (обычно `250px`).
*   **KaTeX:** Для математических формул принудительно задается `max-width: 100%` и `overflow-x: auto`, чтобы длинные формулы можно было прокрутить свайпом, не разрывая ширину всей страницы.

## 6. Анимации
Интерфейс должен быть "живым", но не навязчивым:
*   Микро-анимации при наведении на кликабельные элементы (`250ms cubic-bezier`).
*   Анимация `fadeInUp` (появление снизу-вверх) при открытии модальных окон и переключении вкладок.
*   Импульсная анимация `optionPulse` при выборе ответа в тесте.
*   Плавно переливающиеся цветовые "блобы" (пятна) на фоне главной страницы (через `body::before` и `body::after` с размытием `blur(80px)`).

## 7. UX-требования
*   Минимум лишних кликов. Доступ к настройкам скрыт в `settings-drawer`, разворачивающемся без перезагрузки страницы.
*   При большом количестве предметов используется "Живой поиск" в шапке (Toolbar), чтобы не скроллить весь список.
*   Глобальные действия вынесены отдельно и визуально отделены от контента (например, отступ `margin-bottom: 40px`).
